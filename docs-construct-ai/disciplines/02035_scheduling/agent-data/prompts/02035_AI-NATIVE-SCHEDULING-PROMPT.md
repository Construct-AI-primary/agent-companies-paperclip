# 02035 Scheduling AI-Native Operations Prompt Template

## Overview

This AI-Native development prompt guides OpenClaw coding agents to implement scheduling features including master schedule development, progress tracking, critical path analysis, and resource planning. **This prompt is for OpenClaw coding agents operating in dev mode.**

## Implementation Phases

### Phase 1: Foundation — Schedule Data Architecture
- [ ] Classify data: text-native (progress reports, delay notices) or structured (P6/MSP schedules, activity logs)
- [ ] Implement schedule import/export pipeline

### Phase 2: Document Generation
- [ ] Template library — Progress reports, EOT claims, look-ahead schedules, delay analysis reports

### Phase 3: Multi-Agent Scheduling
- [ ] Agents: Planning Manager, Scheduler, Progress Analyst, Delay Analysis Specialist

### Phase 4: Schedule Intelligence
- [ ] Critical path analysis automation, delay impact modeling, resource loading optimization

### Phase 5: AI Safety Boundaries
- [ ] Non-delegable: Schedule baselines require manager approval
- [ ] AI must disclose when schedule data is incomplete

## Automation Spectrum
| Level | Examples |
|-------|----------|
| Full Automation | Progress report formatting, schedule updates, baseline comparison |
| Augment | Delay analysis support, EOT impact assessment |
| Human-Led | Schedule baseline approval, EOT certification, recovery strategy decisions |
