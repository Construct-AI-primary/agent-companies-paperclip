# 02050 Information Technology Glossary

## Key Terms

### Core IT Concepts

**System Architecture** — The structural design of software systems, including components, their relationships, and the principles governing their design and evolution.

**Error Tracking** — The systematic process of detecting, logging, classifying, and resolving software errors and exceptions in production and development environments.

**State Management** — The technique of maintaining and updating application state across user interactions, page navigation, and data updates in client-side applications.

**Row Level Security (RLS)** — A database security mechanism that restricts user access to specific rows within tables based on user attributes, roles, or session context.

**RAG (Retrieval-Augmented Generation)** — An AI pattern that combines vector database retrieval with LLM generation to provide contextually relevant, accurate responses.

**Prompt Engineering** — The practice of designing and optimizing input prompts for AI/LLM systems to produce desired outputs consistently.

**Template Engine** — A software component that combines templates with structured data to produce documents, pages, or reports dynamically.

**API (Application Programming Interface)** — A set of protocols, tools, and definitions for building software applications that specifies how components should interact.

**AST (Abstract Syntax Tree)** — A tree representation of the abstract syntactic structure of source code, used for static analysis and code transformation.

### Error Management

**Error Discovery** — The automated process of identifying previously unknown errors or error patterns in system logs, user reports, and monitoring data.

**Error Classification** — The process of categorizing errors by type, severity, component, and impact to enable appropriate response prioritization.

**Stack Trace** — A report of the active stack frames at a specific point in time during program execution, used for debugging and error analysis.

**Mean Time to Resolution (MTTR)** — The average time required to resolve a reported error or incident from detection to resolution.

**Anomaly Detection** — The identification of unusual patterns, behaviors, or data points that deviate significantly from expected norms.

### AI Integration

**LangChain** — A framework for developing applications powered by language models, providing tools for chaining LLM calls and managing context.

**LLM (Large Language Model)** — A deep learning model trained on massive text corpora capable of understanding and generating human-like text.

**Vector Database** — A database optimized for storing and querying high-dimensional vectors, enabling semantic search and similarity matching.

**Embedding** — A dense vector representation of text, code, or data that captures semantic meaning for use in machine learning models.

**Agent Orchestration** — The coordination of multiple AI agents to accomplish complex workflows through structured handoffs and data exchange.

**Chatbot** — A conversational AI system that interacts with users through natural language, often integrated with backend systems and knowledge bases.

### Development Operations

**Code Scaffolding** — The automated generation of basic code structure, boilerplate, and patterns from templates or specifications.

**Static Analysis** — The analysis of source code without executing it, to identify potential bugs, security issues, and code quality problems.

**CI/CD (Continuous Integration/Continuous Deployment)** — A DevOps practice of automatically building, testing, and deploying code changes through an automated pipeline.

**Version Control** — The management of changes to source code, documents, and configurations through systems that track and merge changes.

**Data Pipeline** — A system for ingesting, transforming, and loading data from source systems to target systems for analysis and use.

### Security & Access

**Authentication** — The process of verifying the identity of a user, service, or system before granting access to resources.

**Authorization** — The process of determining what actions or resources an authenticated user or service is permitted to access.

**Encryption** — The process of converting data into a coded form to prevent unauthorized access during storage or transmission.

**Audit Trail** — An immutable chronological record of system activities, changes, and access events for security and compliance purposes.

**Access Control** — The selective restriction of access to resources based on identity, role, or attributes.

### AI-Native Operations

**Text-Native Input** — Data that is inherently text-based and can be directly processed by LLMs (documentation, code, logs, configs).

**Code-Native Input** — Source code files that require AST parsing and static analysis before LLM processing.

**Structured Data** — Data organized in a defined schema (JSON, database records, API payloads) suitable for direct programmatic processing.

**Provenance Tracking** — Tagging data-derived values with their source, timestamp, and processing context for audit and verification.

**Automation Spectrum** — Classification of tasks by automation level: Full Automation, Augment AI+Human, Human-Led AI-Informed, Human-Led Only.

**Multi-Agent Orchestration** — The coordination of multiple specialized AI agents through structured workflows with defined handoffs and data formats.
