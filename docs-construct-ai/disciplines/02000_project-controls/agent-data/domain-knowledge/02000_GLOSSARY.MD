# Project Controls Glossary

## Key Terms

### critical path
The longest sequence of dependent activities in the project network that determines the minimum project duration. Any delay on a critical path activity delays the project completion. Activities on the critical path have zero total float.

### float (slack)
The amount of time an activity can be delayed without affecting the project completion date (total float) or the start of its successor activity (free float). Negative float indicates the project is behind schedule.

### baseline schedule
The approved version of the project schedule used as a basis for measuring performance. Any changes to the baseline must go through a formal change approval process.

### earned value (EV)
The value of work performed expressed in terms of the approved budget assigned to that work. Also called budgeted cost of work performed (BCWP). Used to measure project performance objectively.

### planned value (PV)
The authorized budget assigned to the work scheduled to be completed by a specified date. Also called budgeted cost of work scheduled (BCWS). Used to measure schedule performance.

### S-curve
A graph showing cumulative project values (planned value, earned value, actual cost) plotted over time, typically forming an S-shape because projects start slowly, accelerate in the middle, and tail off toward completion.

### SPI (Schedule Performance Index)
A measure of schedule efficiency calculated as EV ÷ PV. An SPI of 1.0 means the project is exactly on schedule; < 1.0 means behind schedule; > 1.0 means ahead of schedule.

### CPI (Cost Performance Index)
A measure of cost efficiency calculated as EV ÷ AC. A CPI of 1.0 means the project is exactly on budget; < 1.0 means over budget; > 1.0 means under budget.

### work breakdown structure (WBS)
A hierarchical decomposition of the total scope of work to be carried out by the project team. Each level represents an increasingly detailed definition of the scope. The lowest level contains work packages.

### risk register
A living document that records all identified risks, their assessments (probability, impact, risk score), planned mitigation actions, risk owners, and current status.

### Monte Carlo simulation
A quantitative risk analysis technique that runs thousands of iterations of the project schedule or cost model using random values drawn from probability distributions, producing a probability distribution for project completion date or total cost.

### look-ahead schedule
A short-term schedule (typically 3 to 6 weeks ahead) developed from the master schedule to plan work in detail at the workface level. Includes resource allocation, material availability, and inspection requirements.

### delay analysis
The systematic evaluation of events or circumstances that caused a project to be delayed, determining entitlement to an extension of time and associated costs.
