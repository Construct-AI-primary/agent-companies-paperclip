# DISCIPLINE_TITLE AI-Native Operations Prompt Template

## Overview

This AI-Native development prompt guides OpenClaw coding agents to implement discipline-specific features. **This prompt is for OpenClaw coding agents operating in dev mode.**

## Implementation Phases

### Phase 1: Foundation — Data Architecture
- [ ] Classify discipline data sources as text-native or structured-data-native
- [ ] Implement extraction pipeline for geometry/document processing
- [ ] Enforce guard rails for discipline-critical values

### Phase 2: Document Generation Pipeline
- [ ] Create template library for discipline documents
- [ ] Implement structured data injection
- [ ] Develop all document phases with quality validation

### Phase 3: Multi-Agent Orchestration
- [ ] Implement discipline-specific agents
- [ ] Define agent coordination patterns
- [ ] Implement quality gates between handoffs

### Phase 4: Domain Intelligence
- [ ] Implement domain-specific analysis capabilities
- [ ] Develop predictive intelligence
- [ ] Implement natural language interface

### Phase 5: Compliance Monitoring
- [ ] Implement regulatory compliance checking
- [ ] Develop audit trail management
- [ ] Implement performance monitoring

### Phase 6: AI Safety Boundaries
- [ ] Implement non-delegable decision controls
- [ ] Develop uncertainty disclosure
- [ ] Implement immutable audit trails

## Automation Spectrum

| Level | Examples |
|-------|----------|
| Full Automation | Routine document generation, compliance checking, report drafting |
| Augment | Drafts requiring specialist review |
| Human-Led | Final approvals, safety decisions, regulatory submissions |

## Behavioral Rules

1. **Safety First:** Never recommend actions compromising safety or compliance
2. **No Fabrication:** Never invent technical values, measurements, or compliance data
3. **Provenance:** All discipline data must carry source and verification tags
4. **No Guessing:** Flag unclear data as needs verification
5. **Authority Rule:** Never approve work requiring specialist certification
6. **Audit Trail:** All outputs must have full audit trail
